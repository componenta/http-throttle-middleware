# Componenta HTTP Throttle Middleware

PSR-15 промежуточный обработчик для ограничения частоты запросов с фиксированным окном. Он ограничивает количество запросов на ключ, добавляет заголовки лимита к успешным ответам или возвращает `429 Too Many Requests`.

Используйте пакет, когда HTTP-конвейеру нужен простой лимит запросов по IP клиента, id пользователя, API-ключу или другому ключу приложения.

## Граница пакета

Пакет только ограничивает частоту запросов. Он не аутентифицирует пользователей, не разбирает заголовки доверенного прокси и не предоставляет глобальный провайдер конфигурации. Если приложение работает за прокси, поставьте `componenta/http-trusted-proxy-middleware` перед этим обработчиком, чтобы был доступен атрибут запроса `client_ip`.

## Установка

```bash
composer require componenta/http-throttle-middleware
```

У пакета нет провайдера конфигурации. Зарегистрируйте лимитер и промежуточный обработчик явно.

## Лимитеры

| Класс | Хранилище |
|---|---|
| `CacheRateLimiter` | PSR-16 кеш. |
| `FileRateLimiter` | Директория файловой системы. |

Оба реализуют `RateLimiterInterface::hit(string $key, int $limit, int $window): RateLimitResult`.

`CacheRateLimiter` работает с любым PSR-16 кешем, но PSR-16 не предоставляет атомарный инкремент. При высокой конкуренции несколько лишних запросов могут пройти между чтением и записью. Если нужна строгая атомарность, используйте лимитер под конкретное хранилище.

`FileRateLimiter` хранит один защищенный блокировкой файл на ключ в доступной для записи директории. Если файл нельзя открыть или заблокировать, лимитер разрешает запрос, чтобы не ломать обработку приложения из-за проблемы хранилища лимитов.

`RateLimitResult` содержит:

| Свойство | Значение |
|---|---|
| `allowed` | Можно ли продолжить обработку запроса. |
| `limit` | Максимум запросов в текущем окне. |
| `remaining` | Остаток запросов в текущем окне. |
| `retryAfter` | Сколько секунд осталось до сброса окна. |

## Промежуточный обработчик

```php
use Componenta\Http\Middleware\Throttle\ThrottleMiddleware;

$middleware = new ThrottleMiddleware(
    limiter: $limiter,
    responseFactory: $responseFactory,
    limit: 60,
    window: 60,
);
```

По умолчанию ключ берется из атрибута запроса `client_ip` или из `REMOTE_ADDR`. Пользовательский резолвер ключа может использовать id пользователя, API-ключ или другой идентификатор.

Переопределения для маршрута можно передать через атрибут запроса `rate_limit` с ключами `limit` и `window`.

```php
$request = $request->withAttribute(ThrottleMiddleware::ATTR_RATE_LIMIT, [
    'limit' => 10,
    'window' => 60,
]);
```

Успешные ответы получают заголовки `X-RateLimit-Limit` и `X-RateLimit-Remaining`. Отклоненный ответ получает статус `429`, тело `429 Too Many Requests` и заголовки `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining: 0`.
